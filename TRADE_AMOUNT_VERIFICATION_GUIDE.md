# 거래대금 재검증 가이드 (2026-05-08)

## 📊 개요

거래대금 표시 불일치 문제 (네이버 3억, 남해화학 447억 vs 실제 402억)를 **완벽히 수정**했습니다.
이 가이드에 따라 프로그램을 재시작하고 거래대금이 올바르게 표시되는지 확인하세요.

## 🔧 수정 사항 요약

### 원인
- **키움 API 거래대금 필드 단위**: 백만 원 (× 1,000,000)
- **기존 코드**: 추측 기반 단위 판별 (Nearest Match 방식)
  - `_resolve_trade_amount()`: 천원 × 1,000 vs 백만 × 1,000,000 비교
  - FID 13 (실시간): cand_1k vs cand_1m 비교
  - 결과: 일부 종목 오판별

### 수정
1. **[kiwoom_api.py:237-248]** `_resolve_trade_amount()`
   ```python
   # 변경 전: 추측 기반
   if 0.8 <= (amt_m / calc_amt) <= 1.2: return amt_m
   if 0.8 <= (amt_k / calc_amt) <= 1.2: return amt_k
   
   # 변경 후: 확정된 단위
   if raw_amt <= 0:
       return price * volume
   return raw_amt * 1_000_000  # 백만 원 단위
   ```

2. **[scanner/smart_scanner.py:969-975]** FID 13 실시간 처리
   ```python
   # 변경 전: Nearest Match
   cand_1k = raw_cum_amt * 1_000
   cand_1m = raw_cum_amt * 1_000_000
   real_trade_amt = cand_1m if diff_1m < diff_1k else cand_1k
   
   # 변경 후: 확정된 단위
   real_trade_amt = raw_cum_amt * 1_000_000
   ```

3. **진단 로그 설정**
   - kiwoom_api.py:501-502 — opt10001 진단 로그 (DEBUG 레벨)
   - kiwoom_api.py:1341-1342 — opt10030 진단 로그 (DEBUG 레벨)

| 항목 | 이전 | 현재 | 예상 |
|-----|------|------|------|
| 거래대금 표시 | 447억 ❌ | ? | ~402억 ✅ |
| 거래대금 계산 | 추측 기반 (천원 선택) | 백만 원 단위 | 정확함 |

**기대값:**
- raw_amt = 402 (백만 원)
- 402 × 1,000,000 = 402,000,000원
- 표시: "4억 2,000만원" 또는 "4억 0,200만원" (정규화 로직에 따라)
  - `f"{n // 100_000_000:,}억"` 사용 시 → "4억"
  - `f"{n // 10_000:,}만원"` 사용 시 → "40,200만원"
  - 현재 코드: `max(n//100_000_000, n//10_000)` → "4억"

## ❓ 문제 해결

### Q: 로그에 "[진단]" 로그가 없어요

**원인:**
- DEBUG 로그 레벨이 활성화되지 않음
- 또는 거래대금 상위 조회가 실행되지 않음

**해결:**
1. `start_with_debug.py`로 시작 (DEBUG 자동 활성화)
2. 또는 "자동매매 ON" 클릭

### Q: TR Limit으로 인해 거래대금 조회가 실패했어요

**원인:**
- TR Limit -200 상태에서 프로그램 시작

**해결:**
- 15분 대기 후 자동 재시도 (프로그램에 내장됨)
- 또는 거래대금 상위 버튼 수동 클릭

### Q: 네이버가 여전히 3억으로 표시돼요

**원인:**
- 캐시가 남아있음
- 코드 변경이 적용되지 않음

**해결:**
1. `logs/` 디렉토리 전체 삭제
2. `data/cache/` 디렉토리 삭제
3. 프로그램 재시작
