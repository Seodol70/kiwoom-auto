## 🎯 매수 신호 타입

### 1️⃣ BREAKOUT (단순 돌파 신호)

**상태:** 🔵 필터링용 (단독 매수 불가)

**조건:**
```
✓ 전일종가 대비 3% 이상 상승 (2026-04-03 강화: 2%→3%, 가짜 돌파 방지)
✓ 당일 거래량 있음 (거래량 체크)
```

**목적:** 기초 여과 (BREAKOUT 통과 → JDM 추가 평가)  
**핵심:** 호가 몇 개 먹은 수준의 가짜 신호 배제, 의미 있는 움직임만 포착

**예시:**
```
전일종가: 10,000원
현재가: 10,300원 이상 필요 (기존 10,200 → 상향)
거래량: > 0 필요
```

### 2️⃣ JDM_ENTRY (실제 매수 신호) ✅

**상태:** 🟢 매수 신호 (5가지 조건 모두 충족 필수)

#### 📋 조건 상세

| # | 조건 | 기준 | 설명 |
|---|------|------|------|
| 0 | **수급 절대치** | 거래대금 **상위 100위** 이내 | 2026-04-21 완화: 금액 제한 철폐, 상위 100위 이내 |
| 1 | **VWAP 생존 게이트** | **현재가 > VWAP * 0.995** | 2026-05-05 완화: 학습을 위해 -0.5% 버퍼 허용 |
| 2 | **MA 정배열 및 이격** | MA7 > MA15 교차 + **이격 ≥ 0.15%** | 1분봉 기준, Whipsaw 방지용 정배열 확인 |
| 3 | **거래량 급증** | 직전 10분 평균 × **1.2배 이상** | 2026-04-21 완화: 1.5배→1.2배 (태동기 포착) |
| 4 | **체결강도** | **110% 이상** | 2026-05-05 완화: 120% → 110% (공격적 진입) |
| 5 | **AI 스마트 판단** | **AI 스코어 > 0.3** | 2026-05-05 완화: 0.4 → 0.3 (학습 데이터 확보 모드) |
| 6 | **RSI** | **40~80% 범위** | 2026-05-05 완화: 45% → 40% (바닥권 탈출 포착) |

> **💡 스코어링 보너스 및 AI 가점**
> - **VWAP**: 현재가가 VWAP 위에 있어야만 AI 판단이 시작됨 (안전 펜스)
> - **MTF (Multi-Timeframe)**: 15분/60분봉 추세가 정배열일 경우 AI 가점 부여
> - **호가 잔량**: 매도잔량이 매수잔량보다 많을 때(물량 흡수 중) 돌파 확률 가점
> - **캔들 패턴**: 꼬리가 짧고 몸통이 긴 양봉 출현 시 가점

#### ⚙️ 기술 파라미터

```python
# scanner/smart_scanner.py SmartScannerConfig
breakout_ratio = 0.03                  # BREAKOUT 기준 (%) — 2026-04-03 강화: 1%→3%
entry_start_time = 09:00:00
entry_end_time = 14:30:00              # 2026-04-03: 11:00→14:30 확장
volume_1min_surge_mult = 1.2           # 거래량 배수 (직전 10분 평균 대비) - 2026-04-21 완화

jdm_ma_long = 15                       # 장기 MA 기간
jdm_min_ma_spread_abs = 30             # [deprecated] MA 이격 최소값 (원)
jdm_ma_spread_pct = 0.15               # MA 이격 비율(%) — 2026-04-21 하향
jdm_rsi_entry_min = 45.0               # RSI 진입 하한 (시간대별 최소치) - 2026-04-21 하향

min_trade_amount = 0                   # 최소 거래대금 제한 철폐 (2026-04-21)
min_daily_rank = 100                   # 거래대금 상위 100위 (2026-04-21 완화)

# config.py RISK
max_change_pct = 22.0                  # 등락률 상한(%) — 2026-04-03 확장: 15→22%
```

## 📈 신호 발생 예시

### ✅ 매수 신호 발생

```
종목: 삼성전자(005930)
시간: 10:30

조건 체크:
✓ 진입 시간: 10:30 ∈ [09:00, 14:30]
✓ 거래량: 현재 1분 20,000주 > 10분평균 13,000 × 1.5 (19,500)
✓ 체결강도: 125% ≥ 120%
✓ MA: MA7(7,280) > MA15(7,250), 이격 30원 = 조건 충족
✓ RSI: 65% ∈ [60, 70]

결과: 🟢 매수 신호 발생!
→ order_mgr.buy() 실행
```

### ❌ 신호 거부 예시

```
종목: 대한해운(005880)
시간: 11:00

조건 체크:
✓ 진입 시간: 11:00 ∈ [09:00, 14:30]
✗ 거래량: 현재 8,000주 < 10분평균 7,000 × 1.5 (10,500) ← 미충족!

결과: ❌ 신호 거부 (거래량 미달)
→ 다른 조건 체크 안 함 (AND 조건)
```

## 💰 포지션 관리

### 손절 (Stop Loss)
```
기준: -2.0% (평단가 대비)
실행: _auto_sell_by_pnl() 1분 주기
대상: qty_buy_today_app > 0 (앱 신호 + 이전 세션 포지션)
```

### 익절 (Take Profit)
```
기준 (동적):
  09:00 ~ 10:00: +1.5%
  10:00 ~ 11:00: +2.0%
  11:00 ~ 14:30: +3.0% (원래 설정)

실행: _auto_sell_by_pnl() 1분 주기
```

### Time-Cut (강제 매도)
```
조건: 진입 후 40분 경과 + 수익률 < 0.5%
목적: 슬롯 효율화 (수익성 낮은 포지션 빠른 청산)
```

## ⚠️ 안전 장치

### 최대 보유 종목 수
```
제한: 7개 (config.STRATEGY["max_positions"])
로직: 신호 수신 시 보유 + 대기 >= 7이면 신호 무시
파일: ui/main_window.py line 1624-1628
```

### 중복 주문 방지
```
✓ 같은 종목 대기 중: 신호 무시
✓ 같은 종목 보유 중: 신호 무시
✓ 최대 수량 도달: 신호 무시
```

### 등락률 상한
```
제한: 15% 이상 급등은 감시 대상에서 제외
이유: 과열 종목 리스크 회피
파일: config.RISK["max_change_pct"]
```

## 📝 주의사항

1. **모든 조건은 AND** (하나라도 미충족 → 신호 거부)
2. **1분봉 기반** (5분, 일봉이 아님)
3. **실시간 가격** (지연 없음)
4. **모의투자 환경** 최적화 (거래량 완화)
