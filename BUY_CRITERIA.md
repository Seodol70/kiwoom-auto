## 🎯 매수 신호 타입

### 1️⃣ BREAKOUT (단순 돌파 신호)

**상태:** 🔵 필터링용 (단독 매수 불가)

**조건:**
```
✓ 전일종가 대비 3% 이상 상승 (2026-04-03 강화: 2%→3%, 가짜 돌파 방지)
✓ 당일 거래량 있음 (거래량 체크)
```

**목적:** 기초 여과 (BREAKOUT 통과 → JDM 추가 평가)  
**핵심:** 호가 몇 개 먹은 수준의 가짜 신호 배제, 의미 있는 움직임만 포착

**예시:**
```
전일종가: 10,000원
현재가: 10,300원 이상 필요 (기존 10,200 → 상향)
거래량: > 0 필요
```

---

### 2️⃣ JDM_ENTRY (실제 매수 신호) ✅

**상태:** 🟢 매수 신호 (5가지 조건 모두 충족 필수)

#### 📋 조건 상세

| # | 조건 | 기준 | 설명 |
|---|------|------|------|
| 0 | **수급 절대치** | 거래대금 상위 50위 OR **500억 이상** | 2026-04-03 강화: 300억→500억 (호가창 얇은 종목 배제) |
| 1 | **진입 시간** | 09:00 ~ 14:30 | 야간 저유동성 제외, 장 마감 30분 전 차단 |
| 2 | **거래량 급증** | 직전 10분 평균 × **1.5배 이상** | 1분봉 거래량 기준, 수급 강도 |
| 3 | **체결강도** | **120% 이상** (+ 최종 재확인) | FID 20, 매수세 우위 확인 + JDM 직전 재확인 추가 (2026-04-03) |
| 4 | **MA 골든크로스** | MA7 > MA15 **교차** + **이격 ≥ 0.3%** | 1분봉 기준, 종목단가 독립적 (30원→비율) |
| 5 | **RSI** | **60~80% 범위** | 2026-04-03 확장: 70→80% (강한 주도주 포착) |

#### ⚙️ 기술 파라미터

```python
# scanner/smart_scanner.py SmartScannerConfig
breakout_ratio = 0.03                  # BREAKOUT 기준 (%) — 2026-04-03 강화: 1%→3%
entry_start_time = 09:00:00
entry_end_time = 14:30:00              # 2026-04-03: 11:00→14:30 확장
volume_1min_surge_mult = 1.5           # 거래량 배수 (직전 10분 평균 대비)
min_chejan_strength = 120.0            # 체결강도 하한 (%) + JDM 직전 재확인 추가

jdm_min_ma_spread_abs = 30             # [deprecated] MA 이격 최소값 (원)
jdm_ma_spread_pct = 0.3                # MA 이격 비율(%) — 2026-04-03 추가
jdm_rsi_entry_min = 60.0               # RSI 진입 하한

min_trade_amount = 50_000_000_000      # 최소 거래대금 500억 (2026-04-03 강화: 300억→500억)

# config.py RISK

```

## 📈 신호 발생 예시

### ✅ 매수 신호 발생

```
종목: 삼성전자(005930)
시간: 10:30

조건 체크:
✓ 진입 시간: 10:30 ∈ [09:00, 14:30]
✓ 거래량: 현재 1분 20,000주 > 10분평균 13,000 × 1.5 (19,500)
✓ 체결강도: 125% ≥ 120%
✓ MA: MA7(7,280) > MA15(7,250), 이격 30원 = 조건 충족
✓ RSI: 65% ∈ [60, 70]

결과: 🟢 매수 신호 발생!
→ order_mgr.buy() 실행
```

### ❌ 신호 거부 예시

```
종목: 대한해운(005880)
시간: 11:00

조건 체크:
✓ 진입 시간: 11:00 ∈ [09:00, 14:30]
✗ 거래량: 현재 8,000주 < 10분평균 7,000 × 1.5 (10,500) ← 미충족!

결과: ❌ 신호 거부 (거래량 미달)
→ 다른 조건 체크 안 함 (AND 조건)
```

## 💰 포지션 관리

### 손절 (Stop Loss)
```
기준: -2.0% (평단가 대비)
실행: _auto_sell_by_pnl() 1분 주기
대상: qty_buy_today_app > 0 (앱 신호 + 이전 세션 포지션)
```

### 익절 (Take Profit)
```
기준 (동적):
  09:00 ~ 10:00: +1.5%
  10:00 ~ 11:00: +2.0%
  11:00 ~ 14:30: +3.0% (원래 설정)

실행: _auto_sell_by_pnl() 1분 주기
```

### Time-Cut (강제 매도)
```
조건: 진입 후 40분 경과 + 수익률 < 0.5%
목적: 슬롯 효율화 (수익성 낮은 포지션 빠른 청산)
```

## ⚠️ 안전 장치

### 최대 보유 종목 수
```
제한: 7개 (config.STRATEGY["max_positions"])
로직: 신호 수신 시 보유 + 대기 >= 7이면 신호 무시
파일: ui/main_window.py line 1624-1628
```

### 중복 주문 방지
```
✓ 같은 종목 대기 중: 신호 무시
✓ 같은 종목 보유 중: 신호 무시
✓ 최대 수량 도달: 신호 무시
```

### 등락률 상한
```
제한: 15% 이상 급등은 감시 대상에서 제외
이유: 과열 종목 리스크 회피
파일: config.RISK["max_change_pct"]
```
